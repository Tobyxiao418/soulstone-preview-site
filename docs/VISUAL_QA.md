# SoulStone Visual QA Checklist

## Finished bracelet preview

- [ ] Completed design does not look like a tray, plate, or wireframe.
- [ ] Beads are large enough to communicate product value.
- [ ] Transparent cutout assets are used in finished mode.
- [ ] No dark square image edges around beads.
- [ ] Bracelet form is wearable / oval, not a perfect diagram circle.
- [ ] Shadows and highlights make the piece feel physical.
- [ ] Checkout preview matches the design preview quality.
- [ ] Mobile layout keeps bracelet visible and tappable.

## Homepage alignment

- [ ] Finished preview feels visually related to the homepage hero product image.
- [ ] Background surface is warm, editorial, and jewelry-like.
- [ ] Copy supports purchase intent, not just design play.

## Regression checks

- [ ] Stone picker still works.
- [ ] Presets still render.
- [ ] Complete Design opens modal.
- [ ] Admin/order preview still reads localStorage payload.
- [ ] `node --check js/bracelet-renderer.js` passes.
