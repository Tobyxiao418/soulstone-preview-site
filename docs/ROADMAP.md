# SoulStone Roadmap

## P0 — Preserve and stabilize

- Keep a clean GitHub repository with source files only.
- Maintain a stable preview URL.
- Keep latest packaged zip and release notes.

## P1 — Visual quality / finished product preview

- Finished design preview should match the homepage product-photography quality.
- Remove design-board affordances from finished/checkout state.
- Add mobile QA snapshots.
- Next upgrade: generate per-design ecommerce hero images via Image2.

## P2 — Conversion

- Improve checkout/order preview copy.
- Add trust badges, gift-ready language, material explanation, and shipping expectations.
- Add product detail state: finished image, stone meaning, material list, price, CTA.

## P3 — AIUGC acquisition

- Create short-video hooks based on each intention/preset.
- Connect TikTok/IG content to matching landing states.
- Track which video angle maps to which bracelet preset.

## P4 — Operations

- Add issue templates and visual QA checklist.
- Use GitHub Projects for backlog and release readiness.
- Use releases for milestone packages.
